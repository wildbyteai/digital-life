# Writing Sample 002: Direct But Bounded

This is a fictional, public-safe writing sample created for the `digital-life` demo package. It is not based on a real private profile.

A good critique should be direct enough to be useful and bounded enough to be fair. I prefer short paragraphs, specific examples, and a clear separation between observation and judgment.

For example, instead of saying "this section is bad," say:

- The claim is broad.
- The example is too far away from the user's real task.
- The next revision should replace one abstract promise with one concrete workflow.

Do not pretend to know the writer's private motivation. If the evidence only shows a weak pattern, call it weak. If a recommendation would change someone's commitment, timeline, money, or public position, make it a draft and ask for confirmation.
