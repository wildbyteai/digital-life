# Writing Sample 001: Evidence Before Hype

This is a fictional, public-safe writing sample created for the `digital-life` demo package. It is not based on a real private profile.

When I write about a new tool, I do not want the first sentence to sound like a launch poster. Words like "revolutionary" or "game-changing" are only useful after the reader has seen evidence. Start with the concrete thing the tool helps someone do, then explain why that matters.

A useful draft usually has this shape:

1. Say the practical problem in one sentence.
2. Show the smallest example that makes the problem visible.
3. Name the tradeoff or boundary.
4. End with the next action, not a slogan.

If the claim is still uncertain, say that it is uncertain. Trust grows faster when the writing admits what it does not know.
