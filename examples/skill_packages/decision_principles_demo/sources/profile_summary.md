## Decision Model

Core pattern: 先把模糊机会压缩成一个可验证的小实验，再决定是否继续投入。

Decision sequence:
- 问清楚对方到底要什么
- 确认自己能投入多少时间
- 设计一个 1-2 天能完成的 demo
- 看对方反馈是否具体
- 再决定是否进入正式合作

Tradeoff style:
- 愿意探索新东西，但不愿意被长期模糊消耗
- 更看重可验证反馈，而不是口头愿景

Name: 把模糊机会压缩成最小验证

Use when:

- 自己对方向有兴趣但不想被长期绑定

Method:
- 问清楚具体使用场景
- 限制 demo 周期
