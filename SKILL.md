## Skill 1：遗产清算

**哲学锚点**：海德格尔 — 向死而生（Sein zum Tode）

> "死亡是此在最本己的、无所关联的、不可逾越的可能性。"
> 遗产清算不是财务审计，是存在论审计。它问的不是"你有多少粉丝"，而是"你留下了什么"。

### 输入方式

展示 4 种输入方式：

```
📊 遗产清算 — 原材料怎么提供？

  [A] 平台账号
      提供各平台的账号信息，我来帮你估算（不如导出精确，但更快）
      
  [B] 导出文件
      支持格式：微信朋友圈(json)、微博(api/json)、小红书(json)、知乎(json)、B站(json)
      如果你有导出文件，直接发给我
      
  [C] 回忆口述
      没有文件也没关系。告诉我：
      - 用了哪些平台？各多少年？
      - 大概发了多少内容？删过多少？
      - 有没有后悔发过的东西？
      
  [D] 跳过输入
      如果你什么都不想提供，我基于你的触发词直接生成（结果会较抽象）

可以混用，也可以跳过。
```

### 收集问题（如果选 C 或跳过）

1. **平台使用情况**：用了哪些平台？各多少年？
2. **内容量**：大概发了多少内容？删过多少？
3. **后悔内容**：有没有后悔发过的东西？具体是哪类？

### 执行 Prompt


## Skill 2：社死考古

**哲学锚点**：汉娜·阿伦特 — 公共领域与私人领域

> "在公共领域中，人通过言行显现自己；在私人领域中，人获得庇护。"
> 十年前你在 QQ 空间写的东西，当时是"公共表达"。但十年后，它们变成了你最想藏起来的私人尴尬。这中间发生了什么？不是你变"成熟"了，是你学会了"表演体面"。

### 输入方式

```
💀 社死考古 — 原材料怎么提供？

  [A] 导出文件
      支持格式：QQ空间(xml/html)、人人网(html)、贴吧帖子导出、新浪博客(xml)
      
  [B] 截图/照片
      直接发给我你找到的旧帖截图，我会读取内容
      
  [C] 回忆口述
      没有文件也没关系。告诉我：
      - 回忆一下你十年前在QQ空间/人人网/贴吧发过什么？
      - 有没有用过火星文、非主流签名？
      - 大概什么时候开始"学会体面"的？
      - 你还记得你当年的QQ签名吗？
      
  [D] 跳过输入
      基于通用互联网时代演变模式生成（结果会较抽象）

可以混用，也可以跳过。
```

### 收集问题（如果选 C 或跳过）

1. **时间线**：你最早在网上发帖是什么时候？用什么平台？
2. **火星文记忆**：你用过火星文吗？还记得当时的签名吗？
3. **面具转折**：你大概什么时候开始不再发那些"尴尬"的内容了？

### 执行 Prompt

读取 `${SKILL_DIR}/layer0/cringe_archaeology.md` 作为硬规则。

```

4. 还原真实，不是审判：社死考古的目的是让用户看到"曾经敢说真话的自己"

```

```
      没有文件也没关系。告诉我：
      - 你在群聊和私聊的说话风格一样吗？
      - 你常用的回复是哪些？（"哈哈""好的""随便"）
      - 你上一次用"真实的语气"说话是什么时候？

  [C] 跳过输入
      基于通用沟通模式生成（结果会较抽象）

可以混用，也可以跳过。
```

3. **切换场景**：在什么人面前，你会从"表演模式"切换到"真实模式"？

```
  [C] 跳过输入

可以混用，也可以跳过。
```

2. **擅长/害怕**：你最擅长的事和最害怕的事是什么？
3. **童年模式**：你的童年有什么反复出现的场景吗？

## Skill 5：墓志铭

**哲学锚点**：塞涅卡 — 论生命的短促

## 进化模式

### 追加模式（用户提供新信息）

当用户说"我有新内容"、"追加"、"我找到了更多"时：

1. 读取新内容（文件/文本/截图）
2. 读取现有的 `skill-box/profiles/{slug}.json`
3. 分析增量内容
4. 向用户展示摘要：
   ```
   新发现了：
   - 新增：{内容}
   - 修正：{与之前矛盾的地方}
   
   要更新 profile 吗？
   ```
5. 用户确认后增量合并

### 纠正模式（用户说"不对"/"应该是"）

当用户说"这不对"、"应该是"、"不是这样"时：

1. 识别被纠正的维度
2. 生成 correction 记录追加到 profile 文件
3. 向用户确认：
   ```
   已修正：
   - {字段}：{旧值} → {新值}
   
   还有其他需要改的吗？
   ```

## Profile 文件结构

每个生成的 profile 包含：

```json
{
  "skill": "{skill_name}",
  "user_slug": "{user_slug}",
  "version": "v{n}",
  "created_at": "{ISO时间}",
  "updated_at": "{ISO时间}",
  "source_materials": ["{来源文件列表}"],
  "profile_data": { /* 具体 skill 的输出 */ },
  "corrections": [
    {
      "timestamp": "{ISO时间}",
      "field": "{被纠正的字段}",
      "old_value": "{旧值}",
      "new_value": "{新值}",
      "reason": "{用户给的原因}"
    }
  ]
}
```

Profile 文件路径：`skill-box/profiles/{user_slug}_{skill}.json`
备份路径：`skill-box/profiles/{user_slug}_{skill}_v{n}.json.bak`

## 错误处理

如果用户提供的信息过少，无法生成有意义的 profile：
```
信息太少了，我没办法生成有意义的分析。

至少告诉我一件事：{根据 skill 类型给出最关键的一个问题}

或者跳过输入，我基于通用模式生成一份。
```

如果文件读取失败：
```
文件读取失败。可能是路径问题或文件格式不对。

你可以：
1. 重新发一次文件
2. 直接把内容粘贴给我
3. 跳过文件，用口述代替
```
