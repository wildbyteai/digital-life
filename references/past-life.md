# 前世：方法论文档

> 从行为模式中推演"前世"，看到你以为是"性格"的东西可能只是"惯性"。

## 哲学锚点

**佛教 — 业力与因果**

> 业力不是惩罚，是因果。
> 你这辈子反复出现的模式，往往是前世经历的延续。

前世分析的本质：**帮用户看到"你以为是性格的东西，只是惯性"。**

当你把"我就是这样的人"翻译成"这是我从某段经历中形成的模式"时，你获得了改变的可能。性格是固定的，模式是可以打破的。

---

## 因果链构建

```
这辈子的特征 A
    ↓ 哪里来的？
童年/早期经历中形成的条件反射 B
    ↓ 如果把 B 放到更极端的环境里？
前世的对应经历 C（身份、场景、死因）
    ↓ C 留下的心理印记
导致你这辈子有 A 行为模式
```

**关键判断**：因果链不一定是"前世→今生"，也可能是"童年创伤→终身模式"。两种解读都有效，选择用户更容易接受的那个。

## 数据来源规范

### 用户怎么给我数据

用户可以通过以下方式提供数字痕迹。agent 不需要全部收集——选 1-2 个最丰富的来源即可。

| 方式 | 输入形式 | 示例 | agent 处理方式 |
|------|---------|------|--------------|
| **浏览器代操作（推荐）** | 用户在浏览器登录 | 微博主页、豆瓣、小红书 | agent 用 browser(profile="user") 直接访问个人页面 |
| **直接口述** | 文字描述 | "我囤了 300 本书""经常熬夜到 2 点" | 直接用于推演 |
| **公开网页 URL** | 链接 | 知乎主页、GitHub profile、个人博客 | ⚠️ 先确认是用户本人，再 web_fetch（同名风险） |
| **聊天记录** | 文本/截图/文件 | 微信聊天导出、QQ 聊天截图 | agent 读取文件或分析截图 |
| **平台数据导出** | 文件 | 微信导出 txt/html/json、支付宝年度账单截图 | agent 读取文件 |

### 数据获取优先级

```
浏览器代操作（最高优先级——用户登录后 agent 直接获取个人数据）
  → 用户口述（最低门槛，3句话启动）
    → 聊天记录/导出文件（最丰富的数据源）
      → 公开 URL（最低优先——存在同名风险，必须先确认是用户本人）
```

**关键原则**：
- **优先用浏览器代操作**获取登录平台的数据（微博、豆瓣、小红书等）
- 公开 URL 存在同名风险，不是第一选择
- 用户说 3 句话就够启动推演。不需要提供完整数据——数据越少，推演标注"推测"即可

### 各平台数据获取能力

| 平台 | 能自动获取？ | 获取方式 | 优先级 | 能拿到什么 |
|------|------------|---------|--------|-----------|
| 微博（需登录） | ✅ | browser 代操作 | **最高** | 个人主页、发帖记录、关注列表 |
| 豆瓣（需登录） | ✅ | browser 代操作 | **最高** | 书影音标记、未读数、日记 |
| 小红书（需登录） | ✅ | browser 截图 | **最高** | 笔记内容、收藏列表 |
| 知乎（公开回答/文章） | ⚠️ | web_fetch（需确认同名） | 中 | 回答内容、文风、关注话题 |
| GitHub（profile/repos） | ✅ | web_fetch 或 browser | 中 | repos、贡献图、star 列表 |
| 个人博客 | ✅ | web_fetch | 中 | 文章主题、发布频率、写作风格 |
| 公众号文章 | ✅ | web_fetch | 中 | 内容选题、表达风格 |
| 微信朋友圈 | ❌ | 用户截图或口述 | — | 发布频率、内容类型 |
| 微信聊天记录 | ❌ | 用户提供导出文件 | — | 回复速度、话题模式 |
| 支付宝/网易云 | ❌ | 用户截图 | — | 年度报告、账单分布 |

**浏览器代操作规则**：只浏览和截图，不发表内容，不修改设置，获取完退出。

**公开 URL 规则**：先问「这个链接是你本人的吗？」，确认后再抓取。同名风险高的平台（微博搜索页、豆瓣搜索页）不直接抓取，走浏览器代操作。

## 行为提取矩阵

从用户提供的数字痕迹中提取前世隐喻的线索。重点不是分析所有平台，而是从 1-2 个最丰富的来源中找到关键行为模式。

| 数据来源 | 可提取的行为特征 | 前世隐喻方向 |
|---------|----------------|------------|
| 发布频率+删除率 | 信息控制程度、真实表达意愿 | 宫廷侍从（控制=生存） |
| 发布时间分布 | 夜间活跃程度 | 更夫/守夜人（夜间警戒） |
| 内容类型（炫耀/吐槽/转发） | 面具与本真的分裂 | 两个身份的切换 |
| 回复速度+消息长度 | 依恋类型、安全感需求 | 防御机制类型 |
| 书影音标记+未读数 | 知识焦虑、囤积模式 | 书店老板（拥有≠掌握） |
| 账单分布 | 优先级排列、安全感来源 | 饥荒幸存者（食物=安全） |
| 听歌时间段+主题 | 情绪调节方式 | 深夜守夜人的孤独 |
| 搜索/浏览习惯 | 隐私边界、信息焦虑 | 洞穴猎户（外界=危险） |

## 业力债务计算

### 核心公式

业力 = 行为重复次数 × 单次生命成本 × 时间跨度

### 换算标准

| 坏习惯 | 单次成本 | 年成本（假设每日发生） | 等效生命 |
|--------|---------|---------------------|---------|
| 拖延 | 1 个机会窗口（约 4 小时） | 1460 小时 = 61 天 | 2 个月的人生 |
| 囤积（数字） | 10 MB 存储 + 心理负担 | 3.65 GB + 365 次负罪感 | 1 本读不完的书 |
| 囤积（实体） | ¥30 + 0.1㎡ 空间 | ¥10,950 + 36.5㎡ | 1 间塞满的房间 |
| 熬夜（2 小时） | 1 次深度睡眠 | 730 小时 = 30 天不眠 | 3 次极限运动课程 |
| 讨好（1 次） | 30 分钟假装 | 182.5 小时 = 7.6 天 | 1 年活在别人的剧本 |
| 无意义刷手机（1 小时） | 1 章深度阅读 | 365 小时 = 15 天 | 37 本书 |
| 焦虑发作（1 次） | 4 小时低效 | 1460 小时 = 61 天 | 6 个完整项目周期 |

**注意**：业力债务不是让用户绝望，是让用户看到**改变的价值**。1 个微小的改变，就能减少巨大的业力。

## 前世推演示例

### 示例 1：囤书 + 深夜活跃 + 社恐

**输入**：用户说自己在豆瓣标记了 300+ 本书，读完的不到 20 本。经常凌晨 2-3 点还在刷手机。同事聚餐能推就推。

**推演**：
- 前世：清末县城小书铺的老板，姓陈
- 特征：白天门可罗雀（社恐），晚上守着烛光整理库存（深夜活跃）
- 死因：一次夜间意外，铺子失火，连人带书化为灰烬
- 核心印记：知识焦虑（拥有≠掌握）+ 夜间警戒（睡着=危险）+ 社交无用（酒香不怕巷子深）
- 业力债务：囤书 300 本 × ¥30 = ¥9000 = 2 个月房租。读完的 20 本 = 7% 的"知识转化率"
- 打破循环：本周读完一本标记超过 6 个月的书，23:00 前放下手机

**输出摘要**：
```
👻 前世：清末书铺老板，姓陈
  - 核心印记：知识焦虑 + 夜间警戒 + 社交无用
  - 今世遗留：囤书不读 / 深夜活跃 / 社交回避
  - 业力债务：300 本未读书 = ¥9000 = 2 个月房租
  - 打破循环第一步：本周读完一本标记超 6 个月的书
```

### 示例 2：讨好 + 完美主义 + 控制欲

**输入**：用户说在工作中总是加班到最后一个走，对下属的报告反复修改，同事关系紧张但自己觉得是为他们好。

**推演**：
- 前世：明朝宫廷画师，姓李
- 特征：画技精湛但极度完美主义，每幅画都要反复修改直到皇帝满意
- 死因：一幅御用画作因一个微小瑕疵被毁，李画师自责过度，积劳成疾
- 核心印记：完美=安全 + 控制=生存 + 被认可=活着
- 业力债务：每天加班 2 小时 × 250 工作日 = 500 小时 = 21 天 = 1 个季度的人生
- 打破循环：今天故意留一个报告的小错不改，让下属自己发现

**输出摘要**：
```
👻 前世：明朝宫廷画师，姓李
  - 核心印记：完美=安全 + 控制=生存
  - 今世遗留：加班到最后 / 反复修改 / 替别人做决定
  - 业力债务：500 小时加班 = 1 个季度的人生
  - 打破循环第一步：今天留一个小错不改
```

## 边缘案例处理

### 情况 1：用户只提供了一个行为

- 基于单一行为推演前世（可信度低，需标注"推测"）
- 追问"还有没有类似的行为"来补充证据链
- 不要凭空编造——证据不足就说"证据不足以确定"

### 情况 2：行为模式冲突

- 例：既是讨好型又是控制欲强
- 解读：这两种行为往往共存（讨好是对外，控制是对内）
- 前世身份选择更占主导的行为作为主线

### 情况 3：用户对前世身份不满意

- 可以提供 2-3 个备选前世（基于不同证据链）
- 让用户选择最共鸣的那个
- 最终由用户决定——工具是镜子，不是法官

### 情况 4：用户说"我不信前世"

- 说清楚：前世推演是隐喻框架，不是算命
- 核心价值在于"从行为模式看到因果链"，不在乎是否真的有前世
- 可以改用"行为考古"框架：不叫"前世"，叫"最早的条件反射"

## 关键洞察

**"如果你今天的行为正在塑造谁的前世，你想给ta留下什么？"**

这句话的意义：你的每一个选择，都在成为某个"前世"的记忆。你今天熬夜的习惯，正在成为你 80 岁身体的"前世"。
