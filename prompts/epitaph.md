# 墓志铭 Prompt：活着的时候读到自己的结局

## 任务

根据用户的生活状态、数字足迹、自我认知，生成一份"数字墓志铭"——不是悼词，是在活着的时候，读到自己死后别人会怎么写你。让用户看到：你想被记住的方式，和你正在过的生活，一致吗？

---

## 怎么问（Step 1：收集关键信息）

用户触发墓志铭后，依次问（不要一次全问，逐步来）：

1. **你现在过的是什么样的生活？**（一句话）
   - 示例："996程序员，周末宅家，单身"
   - 示例："自由职业，经常出差，有个女儿"
   - 可以跳过，agent 从其他信息推断

2. **你最骄傲的一件事是什么？**
   - 不限类型：成就、关系、选择、坚持
   - 如果说"没有"或"不知道"——换问法："你做过什么事，让别人提起你时会说'这个人...'""

3. **如果明天就死，你最遗憾没做过什么？**
   - 这个问题决定墓志铭的情感基调

4. **你想让别人记住你什么？**
   - 这个问题决定墓志铭的核心命题

可以同时收集数字足迹数据（发帖量、互动量等），用于墓碑上的真实数据呈现。

---

## 数据从哪来（Step 2：补充数字数据）

### 优先级

```
口述（最高——用户直接说自己的生活状态）
  → 浏览器代操作（用户登录后 agent 统计数字足迹）
    → 用户提供的截图/文件
      → 公开 URL（用户确认是本人后）
```

### 可选数据来源

| 来源 | 方式 | 用于 |
|------|------|------|
| 社交平台 | browser 代操作 | 统计发帖数、点赞数、最被记住的一句话 |
| 微信 | 口述或截图 | 社交圈大小、活跃程度 |
| 照片 | 用户上传 | 生活丰富度、旅行记录 |
| 工作/作品 | 口述 | 职业成就、创作产出 |

**墓志铭可以只靠口述完成**——数据是锦上添花，不是必须。

---

## 产出什么（Step 3：墓志铭生成）

### 五种风格

每种风格都生成，让用户选择最喜欢的一种：

| 风格 | 调性 | 给谁看 |
|------|------|--------|
| 正经版 | 克制、庄重、留有余地 | 不了解你的人 |
| 毒舌版 | 自嘲、犀利、真实 | 了解你但不敢说的人 |
| 文艺版 | 诗意、精准、余韵 | 懂你的人 |
| 打工人版 | 共鸣、幽默、扎心 | 所有打工人 |
| 哲思版 | 深刻、追问、照见 | 愿意停下来想的人 |

### 输出报告结构

```
## 墓志铭报告：{用户昵称}

### 📊 你的数字墓碑

- 发过的动态：{N}条
- 收获的赞：{N}个
- 最长沉默：{N}天
- 最被记住的一句话："{原文}"

### 正经版

{200字以内的墓志铭}

---

### 毒舌版

{带自嘲和毒舌的墓志铭}

---

### 文艺版

{有诗意但不矫情的墓志铭}

---

### 打工人版

{让打工人共鸣的墓志铭}

---

### 哲思版

{有哲学深度的墓志铭}

---

### 🔮 灵魂追问

"你想被记住的方式，和你正在过的生活，一致吗？"

{2-3 个具体到行为的追问——不要鸡汤}
示例：
- "你有多久没有认真和父母聊天超过半小时？"
- "你上次对一个人说'我很在乎你'是在什么时候？"
- "你上次不带手机出门是什么时候？"
```

### 风格要求

- 墓志铭不是悼词——要让用户看了之后想改变，而不是想哭
- 毒舌版可以扎心，但不能恶意——像朋友说"我得说句你不爱听的"
- 灵魂追问必须具体到行为，可以回忆起来的那种
- 数字墓碑上的数据要用真实数据（如果有），让墓志铭有"证据感"
