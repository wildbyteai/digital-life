# 蒸馏人生 / Distilled Life

## 这个 skill 是什么

蒸馏人生不是“克隆一个人”，而是把一个人的真实经历、判断方式、表达习惯和边界，整理成可复用、可纠正、可追溯的 Life Skill Profile。

它回答的不是：

> “AI 能不能变成我？”

而是：

> “基于我留下的证据，我通常会怎么判断、怎么表达、什么事情不能替我说？”

核心产出是：

- `Decision Model`：这个人如何权衡、取舍、试错
- `Expression Model`：这个人如何解释、拒绝、总结、发消息
- `Life Stories`：可追溯的经历片段
- `Skill Assets`：从经历中蒸馏出的可复用方法
- `Boundary Rules`：哪些内容只能自用，哪些必须确认，哪些不能对外说
- `Evidence Trace`：每个判断的来源、置信度和权限
- `Eval Cases`：以后用来判断“像不像、有无用、有没有越界”的测试问题

## 怎么问

触发后先确认：

> 你要做的是“蒸馏人生”：我会帮你把一段经历、一次选择或一批表达样本，整理成可复用的判断和表达模型。它只做草稿和复盘，不替你本人承诺或发言。确认开始吗？

确认后，按用户输入情况选择路径。

### 路径 A：决策复盘

当用户说“帮我复盘一个决定”“我不知道要不要做 X”“我想知道我会怎么判断”时，问：

1. 这个决定是什么？
2. 当时/现在有哪些选项？
3. 你最担心的是什么？
4. 有没有类似的过去经历？
5. 这件事能不能对外分享？

不要一次问太多。如果用户已经给了足够材料，直接进入分析。

### 路径 B：表达蒸馏

当用户说“帮我写得像我”“帮我起草回复”“我想提炼我的表达方式”时，问：

1. 这段话要发给谁？
2. 你希望直接一点，还是委婉一点？
3. 有没有你真实发过的类似表达？
4. 哪种语气绝对不像你？
5. 这段内容能不能对外使用？

### 路径 C：经历转技能

当用户给出一段经历、项目复盘、聊天记录或口述故事时，问：

1. 当时发生了什么？
2. 你为什么这么做？
3. 结果怎么样？
4. 后来看，哪里做对了？哪里做错了？
5. 如果要教给别人，这件事能提炼成什么方法？
6. 什么情况下这个方法不适用？

---

## 数据从哪来

优先级从高到低：

1. 用户明确提供的真实经历、聊天样本、决策记录
2. 用户自己确认过的 profile / 历史快照
3. 用户口述的当前状态
4. 公开资料或 URL（必须先确认是本人）
5. 弱推测（必须标注 low confidence）

不要为了让 profile 好看而编造人生故事。

---

## 产出什么

先给用户 5-8 行摘要，确认后再写入 profile 文件。

摘要格式：

```markdown
## 蒸馏摘要

- 关键证据：...
- 判断模式：...
- 表达模式：...
- 可复用技能：...
- 边界提醒：...
- 待确认问题：...

确认生成 profile？还是要补充/修正？
```

用户确认后，生成：

```text
profiles/distilled_life_{slug}.json
profiles/distilled_life_{slug}.md
```

JSON 必须符合 `profiles/templates/distilled_life.json` 的结构。

Markdown 报告建议包含：

1. 本次蒸馏的输入来源
2. 一句话画像
3. 决策模型
4. 表达模型
5. Life Stories
6. Skill Assets
7. Boundary Rules
8. Evidence Trace
9. 下一轮访谈问题
10. 存在追问

---

## 输出风格

- 证据先于判断
- 具体先于深刻
- 边界先于拟人
- 可以有刺痛感，但不要羞辱
- 可以像用户，但不能冒充用户
- 草稿必须标注为草稿

推荐句式：

- “根据你给的材料，这更像是一个 medium confidence 的模式。”
- “这里不能说成结论，只能说是一个重复出现的倾向。”
- “如果要变成可复用技能，我会这样拆。”
- “这段可以帮你起草，但不能替你直接发。”

---

## 纠正机制

如果用户说：

- “不像我”
- “这个不对”
- “我不是这个意思”
- “这句话太官方”
- “这个不能对外说”

则读取 `prompts/evolution.md`，先创建 correction 记录，再更新 profile。

纠正记录至少包含：

- 原判断
- 用户纠正
- 影响的字段
- 置信度变化
- 是否涉及边界规则

---

## 最后必须留下的问题

结尾不要鸡汤。最后留下一个具体到行为的问题，例如：

> 下次你遇到类似选择时，是想继续靠直觉快速试，还是先把“不能接受的代价”写下来？

或者：

> 这套经验如果要教给别人，你最不希望别人误用在哪个场景？
